Orchestration and Configuration Management With Ansible

It’s hard to set up development environments. It’s hard to configure servers. It’s hard to manage service and application dependencies. It’s even harder to maintain those things in a consistent way over time, especially with a team. It’s hardest to document things.

Configuration management and orchestration are ways of easing the pain. This talk will introduce Ansible, an open source IT automation tool. It will cover some good uses, bad uses, wins, and pitfalls. General strategies for configuration management, applicable to tools like Chef, Puppet, and Salt, will also be discussed.
